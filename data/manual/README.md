# Manual page data format

Each `Page NN.json` file is the repeatable source for one PDF page.

Page ownership:
- Pages 01-12: Shaan.
- Pages 13-24: Veer.
- Pages 25-32: Krish.
- The owner is generated by `OWNER_RANGES` in `scripts/build_workbook.py` and appears in Control Panel, Master List, and each page tab.

Sheet roles:
- Page tabs mirror the scan page and hold line/roll-level working data.
- Master List is the cross-page inventory table. Its scan-derived columns mirror
  the page tabs with formulas; edit the page tab, not Master List.
- Control Panel is workflow only: owner, PDF/page links, status, line count, needs-check count, OK count, and page note.
- Do not add scan totals or inventory totals to Control Panel unless they are needed for workflow.
- Only pages in `VERIFIED_PAGES` in `scripts/build_workbook.py` feed Master List.
- Pages outside `VERIFIED_PAGES` stay as assigned `NEED UPDATING` placeholders until their scan data is verified.

Top-level fields:
- `date`: shown on the page tab and Master List.
- `item`: fabric item name.
- `page_total_pcs` and `page_total_yards`: optional written total from the scanned page.
- `notes`: page-level issue or scan note. This becomes `Master Note`.
- `fabrics`: one entry per written line/design on the scan.

Fabric fields:
- `label`: design/line name shown on the page tab.
- `grid`: roll yard values, written left-to-right/top-to-bottom from the scan.
- `total_amount`: optional written yards for that fabric line.
- `piece_count`: optional written piece count for that fabric line.
- `notes`: line-specific issue. This becomes `Line Note`.
- `needs_review`: optional manual flag when the line should be reviewed even without a note.

Review rules:
- `Needs Check` is dynamic/read-only.
- `OK?` is the only manual line checkbox.
- A line needs check when it has a `Line Note`, `needs_review`, an unclear/missing roll value, or a mismatch between written totals and roll totals.
- Checking `OK?` turns the row green and clears `Needs Check`.
- Sold state, sold date, customer/invoice, roll note, and `OK?` checkboxes are preserved across rebuilds.
